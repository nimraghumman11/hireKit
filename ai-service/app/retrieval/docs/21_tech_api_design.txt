API design best practices for technical interviews: RESTful conventions: proper HTTP methods (GET/POST/PUT/PATCH/DELETE), meaningful resource naming. Versioning strategies: URL versioning (/v1/), header versioning. Pagination: cursor-based vs. offset-based trade-offs. Error handling: consistent error envelope, meaningful status codes. Authentication: JWT vs. session tokens, OAuth2 flows. Rate limiting, idempotency keys for mutating operations, and API documentation (OpenAPI/Swagger).
